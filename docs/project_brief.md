# Project Brief

## Название

Распознавание маршрутов городского транспорта по фото и видео

## Проблема

Проектов по распознаванию маршрутного номера нет (либо очень мало).

Обычное OCR по изображению ошибается, потому что на фото есть посторонние числа, реклама, госномера, номера домов и дорожные знаки.

Наша задача - сделать пайплайн с качественным определением типа маршрутного транспорта и его номера.

## Целевая аудитория

Городские сервисы мониторинга транспорта, учебный CV-проект, пользователи и аналитики городского транспорта.

## Пользовательский сценарий

1. Пользователь загружает фото или видео.
2. Система находит транспорт.
3. Система определяет тип транспорта и номер маршрута.
4. Пользователь видит результат и bounding boxes.

## AI-задача

Вход: фото или видео городского транспорта.

Выход: тип транспорта, номер маршрута, bounding boxes и confidence.

## Почему без ИИ хуже

Маршрутное табло имеет разные шрифты, ракурсы, размеры и условия освещения. Без ИИ невозможно надёжно найти транспорт и отделить номер маршрута от других чисел на изображении.

## Тип AI-адаптации

CV: fine-tune двух YOLO-моделей на собственном датасете + OCR для чтения номера на crop'ах `route_display`.

## Baseline

Предобученная YOLO на COCO без дообучения находит транспорт и возвращает bbox без номера маршрута.

## MVP

Веб-страница для загрузки изображения, FastAPI backend, YOLO №1 для транспорта, YOLO №2 для `route_display` и OCR для распознавания номера маршрута.

## Целевые метрики

- `AP@50 (bus)`
- `AP@50 (trolleybus)`
- `AP@50 (route_display)`
- `OCR accuracy`
- `End-to-end Accuracy`
- `Latency`

## Baseline / MVP / Target

Baseline: готовая YOLO без дообучения находит транспорт, но не распознаёт номер маршрута.

MVP: YOLO №1 ищет транспорт, YOLO №2 ищет `route_display` внутри crop транспорта, OCR читает номер.

Target: больше данных, лучше разметка, устойчивое распознавание номера и поддержка видео.

## Метрики baseline

- `AP@50 (bus)`
- `precision`
- `recall`

## Риски

- мало данных;
- плохое качество фото;
- маленькое или размытое табло;
- несколько транспортных средств на одном изображении;
- OCR может ошибаться на LED-табло.

## Распределение по ролям

AI: датасет, разметка, обучение модели, метрики.

Backend: FastAPI, endpoint для распознавания, интеграция AI.

Frontend: загрузка изображения, отображение результата.

## УТП

Система определяет не только тип транспорта, но и связывает с ним найденное маршрутное табло. Это позволяет распознавать номер маршрута точнее, чем OCR по всему изображению.
