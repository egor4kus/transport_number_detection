# Project Brief

## Название

Распознавание маршрутов городского транспорта по фото и видео

## Проблема

Проектов по распознаванию маршрутного номера нет (либо очень мало).

Обычное OCR по изображению ошибается, потому что на фото есть посторонние числа, реклама, госномера, номера домов и дорожные знаки.

Наша задача - сделать пайплайн с качественным определением типа маршрутного траспорта и его номера.

## Целевая аудитория

Городские сервисы мониторинга транспорта, учебный CV-проект, пользователи и аналитики городского транспорта.

## Пользовательский сценарий

1. Пользователь загружает фото или видео.
2. Система находит транспорт.
3. Система определяет тип транспорта и номер маршрута.
4. Пользователь видит результат и bounding boxes.

## AI-задача

Вход: фото или видео городского транспорта.

Выход: тип транспорта, номер маршрута, bounding boxes и confidence.

## Почему без ИИ хуже

Маршрутное табло имеет разные шрифты, ракурсы, размеры и условия освещения. Без ИИ невозможно надёжно найти транспорт и отделить номер маршрута от других чисел на изображении.

## Тип AI-адаптации

CV: fine-tune YOLO-детектора на собственном датасете + распознавание номера на crop'ах `route_display`.

## Baseline

Предобученная YOLO на COCO для класса `bus` + OCR без дообучения.

## MVP

Веб-страница для загрузки изображения, FastAPI backend, AI pipeline для поиска транспорта/табло и распознавания номера маршрута.

## Целевые метрики

- `mAP@50`
- `Route Display Recall`
- `Route Number Accuracy`
- `End-to-end Accuracy`
- `Latency`

## Baseline / MVP / Target

Baseline: готовая YOLO + OCR без обучения на наших данных.

MVP: YOLO дообучена на наших данных, номер распознаётся с crop'а табло.

Target: больше данных, лучше разметка, улучшенное распознавание номера, поддержка видео.

## Риски

- мало данных;
- плохое качество фото;
- маленькое или размытое табло;
- несколько транспортных средств на одном изображении;
- OCR может ошибаться на LED-табло.

## Распределение по ролям

AI: датасет, разметка, обучение модели, метрики.

Backend: FastAPI, endpoint для распознавания, интеграция AI.

Frontend: загрузка изображения, отображение результата.

## УТП

Система определяет не только тип транспорта, но и связывает с ним найденное маршрутное табло. Это очень амбициозная задача, поэтому аналоги на рынке найти сложно.
