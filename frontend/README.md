# frontend

Минимальный статический frontend без фреймворков и без сборщика.

## Что делает

1. Загружает список моделей из backend.
2. Отправляет изображение и выбранную модель в `POST /predict/image`.
3. Рисует результат поверх исходного изображения на `canvas`.

## Файлы

```text
frontend/
  README.md
  index.html
  src/
    api.js
    annotator.js
    main.js
    styles.css
```

- `index.html` — одна страница с формой, канвасом, списком объектов и raw JSON.
- `src/api.js` — запросы к backend: `GET /models` и `POST /predict/image`.
- `src/annotator.js` — загрузка изображения в браузере и отрисовка рамок.
- `src/main.js` — связывает UI, API и рендер результата.
- `src/styles.css` — простые стили страницы.

## Локальный запуск

Перед запуском frontend backend уже должен работать на `127.0.0.1:8000`.

```bash
cd frontend
python3 -m http.server 3000
```

После этого frontend доступен на:

```text
http://127.0.0.1:3000
```

Локально frontend ожидает backend на:

```text
http://127.0.0.1:8000
```

## Как работает

- При локальном запуске страница открывается на `127.0.0.1:3000` и ходит в backend на `127.0.0.1:8000`.
- При `docker compose` страница открывается на `127.0.0.1:8080` и ходит в backend через `nginx` proxy.
- Для отрисовки используется `canvas`, чтобы рисовать рамки в координатах исходного изображения без дополнительного пересчета.

## Docker Compose

При `docker compose` основная входная точка:

```text
http://127.0.0.1:8080
```

В этом режиме backend отдельно наружу не открыт.

## Что показывает интерфейс

После запроса страница отображает:

- изображение с рамками;
- список найденных `transports`;
- количество `route_displays` для каждого транспорта;
- raw JSON-ответ backend.
