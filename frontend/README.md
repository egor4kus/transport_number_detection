# frontend

Здесь лежит минимальный статический frontend для демонстрации проекта без
фреймворков и без сборщика. Он делает только три вещи:

1. Загружает список моделей из backend.
2. Отправляет изображение и выбранную модель в `POST /predict/image`.
3. Рисует результат поверх исходного изображения на `canvas`.

Текущая структура:

```text
frontend/
  README.md
  index.html
  src/
    api.js
    annotator.js
    main.js
    styles.css
```

Назначение файлов:

- `index.html` — одна страница с формой, канвасом, списком объектов и raw JSON.
- `src/api.js` — запросы к backend: `GET /models` и `POST /predict/image`.
- `src/annotator.js` — загрузка изображения в браузере и отрисовка рамок.
- `src/main.js` — связывает UI, API и рендер результата.
- `src/styles.css` — простые стили страницы.

## Локальный запуск

Сначала поднимите backend из корня репозитория:

```bash
uvicorn backend.app:app --reload --host 127.0.0.1 --port 8000
```

Потом поднимите статический frontend:

```bash
cd frontend
python3 -m http.server 3000
```

После этого откройте:

```text
http://127.0.0.1:3000
```

Почему нужен HTTP-сервер, а не просто открытие файла:

- браузеру удобнее работать с ES-модулями через `http://`;
- так предсказуемее ведут себя запросы к backend;
- не возникает проблем с `file://`-режимом.

## Как работает логика

Страница выполняет следующий поток:

1. При загрузке вызывает `GET /models`.
2. Строит выпадающий список моделей из ответа backend.
3. Пользователь выбирает фото и модель.
4. Фронтенд отправляет `multipart/form-data` на `POST /predict/image`.
5. Получает JSON c `image.width`, `image.height`, `transports[]` и `bbox`.
6. Загружает исходный файл в браузере и рисует его на `canvas`.
7. Поверх изображения рисует рамки и подписи по координатам из ответа backend.

## Почему используется canvas

Для этого проекта `canvas` проще всего:

- backend уже возвращает реальные размеры изображения;
- можно установить `canvas.width` и `canvas.height` равными размерам из ответа;
- рамки рисуются в исходных координатах без дополнительного пересчета;
- затем `canvas` можно просто уменьшить на странице через CSS.

Это особенно удобно, потому что backend в AI-слое уже приводит изображение к
корректной EXIF-ориентации. За счет этого координаты из ответа и картинка на
странице должны совпадать.

## URL backend

По умолчанию frontend пытается обратиться к backend на том же хосте, но на
порту `8000`.

Примеры:

- frontend открыт как `http://127.0.0.1:3000` -> backend ожидается на `http://127.0.0.1:8000`
- frontend открыт как `http://localhost:3000` -> backend ожидается на `http://localhost:8000`

Если позже backend будет жить по другому адресу, можно задать URL через
`<meta name="api-base-url" content="...">` в `index.html`.

## Что показывает интерфейс

После запроса страница отображает:

- изображение с рамками;
- список найденных `transports`;
- количество `route_displays` для каждого транспорта;
- raw JSON-ответ backend.

Сейчас в `GET /models` доступна только `baseline`, но frontend не хардкодит
эту модель и всегда берет список из API.
